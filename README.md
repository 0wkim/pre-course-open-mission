## 미션 개요
- 구현한 미션 이름 : 🏷️우리말 초성 퀴즈

<br />

- 핵심 기능   
1) 두자리의 랜덤한 초성 조합 생성    
2) 해당 제시어(초성)를 제공하고, 사용자의 정답 입력 대기  
3) 30초의 시간 제한  
4) 10초 지나면 첫번째 힌트 제공   
5) 20초 지나면 두번째 힌트 제공  
6) 사용자가 입력한 단어가 실제 사전에 존재하면 정답 처리, 존재하지 않으면 오답 처리

<br />

##  목표
1. Class 및 기능 분리에 익숙해지기
    - 기존 미션에서 진행하던 클래스 및 기능 분리를 더 공부하며, 특히 MVC 패턴에 대한 설계 이해도를 높인다. 

2. static 키워드 활용해보기
    - 정적 메소드의 개념을 이해하고, 유틸리티 로직에 실제로 적용한다.

3. 비동기(async/await) 처리 이해
    - 네트워크 요청 흐름과 await 동작 원리를 명확히 이해한다.

4. 비동기를 활용한 API 연동
    - 기존에는 AI의 도움을 받아 API를 불러오며 비동기적으로 처리했지만, 이번에는 AI의  도움 없이 비동기 처리에 대한 이해를 바탕으로 API를 호출해본다. 

5. 테스트 코드의 구조적 개선 
    - 직전 미션에서는 test 코드를 작성하기 위해 클래스 내부 메소드의 #private 키워드를 수정했었다. 다만, 피드백을 통해 이 방식이 좋지 않다는 것을 깨달았고, 본 미션에서는 내부 메소드의 키워드 수정 없이 test 코드를 작성하도록 한다.

6. 자바스크립트 언어에 대한 추가적인 이해 
    - 사용해본적 없는 다른 언어를 활용해서 기존 미션을 재진행하는 것도 생각해봤지만 자바스크립트에 대한 이해도가 아직 높지 않기 때문에, 이번 미션에서는 자바스크립트에 대한 이해도를 더 높이는 것을 목표로 한다.

## 예외처리 상세
- 초성 생성 : 만약 제시어를 불러오는 데 3분을 초과하는 경우, 재시도한다. 

- 사용자 입력 : 사용자가 아무것도 입력하지 않은 경우, 다시 입력하라는 메시지를 표시한다.

<br/>

## 테스트 상세
1. 랜덤한 음절이 생성되는지 확인한다.

2. API에서 랜덤한 단어가 불러와지는지 확인한다.

3. 제시어 생성 시간이 3분을 넘어갔을 때, 예외처리 메시지가 뜨는지 확인한다.

4. API에서 불러온 단어로 초성 추출이 잘 되는지 확인한다.

5. 10초 뒤, 20초 뒤에 힌트가 잘 나오는지 확인한다.

6. 사용자가 오답을 입력할 경우, 오답에 대한 멘트가 나오는지 확인한다.

7. 사용자가 정답을 입력할 경우, 정답에 대한 멘트가 나오는지 확인한다.

8. 사용자가 아무것도 입력하지 않으면, 예외 메시지가 뜨는지 확인한다.

9. 제시어의 범주에 아무것도 없을 경우, "일반"이라는 값이 들어가는지 확인한다. 

10. 정의 문자열 내부의 태그들이 잘 디코딩되는지 확인한다.
